# Contributing

Contributions that improve accuracy, organization, or coverage are welcome.

## Required information for a new resource

- Repository name and URL
- Original upstream URL, when the resource is a fork
- Category and resource type
- Knowledge level and primary language
- Main benefit
- Recommended research use
- Current license
- Maintenance status
- Verification date

## Acceptance criteria

A resource should have clear value for mechanical engineering, CFD, numerical methods, AI for CFD, reduced-order modeling, FEA, or scientific communication. Avoid redundant entries and describe resources objectively.

## Update procedure

1. Verify the link and upstream project.
2. Review the license and archive status.
3. Update `resources/catalog.yml`.
4. Update `README.md` and relevant learning paths.
5. Run the link-check workflow.
6. Use a descriptive commit message.
